# Быстрое решение проблемы с BOT_TOKEN

## Проблема
Docker Compose не подхватывает токен из `.env` файла.

## Решение

### Шаг 1: Проверьте файл .env

Убедитесь, что файл `.env` существует и содержит токен:

```powershell
# Проверить содержимое
Get-Content .env
```

Должно быть:
```
BOT_TOKEN=ваш_токен_здесь
```

**Важно:** 
- НЕ должно быть пробелов вокруг `=`
- НЕ должно быть кавычек вокруг токена
- НЕ должно быть пустых строк в начале файла

### Шаг 2: Пересоздайте контейнер

```powershell
# Остановите и удалите контейнер
docker-compose down

# Пересоздайте и запустите
docker-compose up -d --build
```

### Шаг 3: Проверьте переменные окружения в контейнере

```powershell
# Проверить, что переменная установлена
docker-compose exec bot env | Select-String BOT_TOKEN
```

Должно показать: `BOT_TOKEN=ваш_токен`

### Шаг 4: Проверьте логи

```powershell
docker-compose logs -f
```

Если токен установлен правильно, ошибка исчезнет.

---

## Альтернатива: Установить токен напрямую

Если `.env` не работает, можно установить токен напрямую:

```powershell
# Остановить контейнер
docker-compose down

# Запустить с токеном
$env:BOT_TOKEN="ваш_токен"; docker-compose up -d
```

---

## Проверка формата .env файла

Правильный формат:
```
BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
```

Неправильные форматы:
```
BOT_TOKEN = токен          # пробелы вокруг =
BOT_TOKEN="токен"         # кавычки
BOT_TOKEN='токен'         # кавычки
# BOT_TOKEN=токен         # закомментировано
```

---

## Если ничего не помогает

1. Удалите контейнер и образ:
   ```powershell
   docker-compose down
   docker rmi k-c-business-lunch-bot-bot
   ```

2. Пересоздайте всё заново:
   ```powershell
   docker-compose build --no-cache
   docker-compose up -d
   ```
